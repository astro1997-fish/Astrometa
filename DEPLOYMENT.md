# Deployment Guide — Cloudflare Pages (frontend) + Railway (backend)

## Overview

| Layer    | Service           | Cost       |
|----------|-------------------|------------|
| Frontend | Cloudflare Pages  | Free       |
| Backend  | Railway Hobby     | ~$6–8/mo   |
| Database | Supabase          | Free tier  |
| Domain   | Cloudflare Registrar | ~$9/yr  |

Deploy **Railway first** so you have the backend URL before configuring the frontend.

---

## Step 1 — Deploy the backend to Railway

1. Go to [railway.app](https://railway.app) and sign in (GitHub login is easiest).
2. Click **New Project → Deploy from GitHub repo** and select this repository.
3. Railway will auto-detect `railway.toml` and use it. No extra config needed for the build.
4. In the Railway project, open **Variables** and add every variable from the list below.
5. Click **Deploy**. Once it goes green, copy the generated URL (e.g. `https://your-app.up.railway.app`). You'll need it in Step 2.

### Required Railway environment variables

```
NODE_ENV=production
PORT=8000

# Supabase
SUPABASE_URL=https://szoowwutahefnjnsevkc.supabase.co
SUPABASE_SERVICE_ROLE_KEY=<from Supabase dashboard → Settings → API>

# Auth / security
SESSION_SECRET=<long random string — 64 chars>
SETTINGS_ENCRYPTION_KEY=<long random string — 64 chars>

# Frontend URL (set to your Cloudflare Pages URL once you know it — see Step 2)
FRONTEND_URL=https://your-site.pages.dev
PRODUCTION_URL=https://your-custom-domain.com   # add later if using a custom domain

# Email (SMTP)
SMTP_HOST=smtp.gmail.com
SMTP_PORT=587
SMTP_SECURE=false
SMTP_USER=<your Gmail address>
SMTP_PASS=<your Gmail app password>
ADMIN_EMAIL=<email to receive admin alerts>

# Bitcoin
BTC_XPUB=<your xpub key>

# Ethereum / EVM
ETH_RPC_URL=<your Alchemy/Infura WebSocket URL — wss://...>

# Push notifications (auto-generated by the app on first start — leave blank initially)
# VAPID_PUBLIC_KEY=
# VAPID_PRIVATE_KEY=
```

> **Tip:** Railway has a "Reference Variable" feature — you can set `FRONTEND_URL=${{RAILWAY_PUBLIC_DOMAIN}}` if your frontend is also on Railway, but for Cloudflare Pages you'll type the URL manually.

## Step 2 — Deploy the frontend to Cloudflare Pages

1. Go to [pages.cloudflare.com](https://pages.cloudflare.com) and sign in.
2. Click **Create a project → Connect to Git** and select this repository.
3. Set **Build configuration**:
   - **Framework preset:** None (or Vite)
   - **Build command:** `cd frontend && npm ci && npm run build`
   - **Build output directory:** `frontend/dist`
   - **Root directory:** `/` (leave as repo root)
4. Under **Environment variables**, add:

```
VITE_SUPABASE_URL=https://szoowwutahefnjnsevkc.supabase.co
VITE_SUPABASE_ANON_KEY=<from Supabase dashboard → Settings → API → anon key>
VITE_API_URL=https://your-app.up.railway.app   ← the Railway URL from Step 1
```

5. Click **Save and Deploy**. Once it finishes, Cloudflare gives you a URL like `https://your-site.pages.dev`.
6. Go back to Railway → Variables and update `FRONTEND_URL` to that Cloudflare Pages URL, then redeploy the Railway service (so CORS accepts it).

## Step 3 — Connect a custom domain (optional)

### Domain on Cloudflare Pages
1. Buy your domain at [Cloudflare Registrar](https://domains.cloudflare.com) — at-cost pricing, ~$9/yr for `.com`.
2. In Cloudflare Pages → your project → **Custom domains** → Add domain.
3. Since the domain is already on Cloudflare, DNS is wired automatically.
4. Update Railway → `PRODUCTION_URL` to `https://yourdomain.com` and redeploy.

### Custom domain on Railway backend (optional)
1. In Railway → your service → **Settings → Domains** → Add custom domain.
2. Add a CNAME record in Cloudflare DNS pointing to the Railway-provided target.
3. Update `VITE_API_URL` in Cloudflare Pages to your Railway custom domain and redeploy the frontend.

## Step 4 — Run the Supabase migration on production

If you haven't already applied the full `supabase/migration.sql` to your Supabase project:

1. Open your Supabase project → **SQL Editor**.
2. Paste the full contents of `supabase/migration.sql` and run it.

## Verify everything works

```bash
# Backend health check (replace with your Railway URL)
curl https://your-app.up.railway.app/health

# Payment capabilities (should show all true after migration)
curl https://your-app.up.railway.app/api/payments/capabilities
```

Both should respond with JSON. If `/health` returns 200, the blockchain listener and BTC monitor are running correctly.

## Ongoing

- **Backend logs:** Railway → your service → **Logs** tab (live streaming).
- **Frontend deploys:** Automatic on every Git push to your default branch.
- **Backend redeploys:** Automatic on every Git push (Railway watches the repo).
- **Supabase backups:** Upgrade to Supabase Pro ($25/mo) once you have real users — free tier pauses after 7 days of inactivity.
